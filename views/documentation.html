DOCUMENTATION

Introduction

This documentation is designed to help craft the correct URL needed to recieve the correctly desired JSON object from the rate my internship website.Below will contain example searches and what the table schema is to provide insight on the type of information recieved and how to get it.

Current Version

The current version for the API is 1.0.0

Authentication

Rate My Internship is an open API at the moment and does not require any authentication to GET the data.

Table Schema


Name Description Type
uid Unique ID of the review Integer
company Reviewed company's name String
location Location of the internship String
position Student's position at the company String
start Beginning date of the internship Date
end End date of the internship Date
intern_type Reviewed company's name String
pay_type Annual or hourly String
pay_amount Income level or hourly rate String
work_desc Description of tasks performed at internship String
overall_rating Overall experience at the company Integer
leadership_rating Leadership of the team or mentor Integer
communication_rating Communication between team members and superiors Integer
flexibility_rating Flexibility or willingness to accomadate student schedules Integer
culture_rating Coprorate culture Integer
internship_recommend Likelihood of recommending the internship to another student integer
company_recommend Likelihood of recommending the company to another student Integer
rating_desc Comments on reviewers ratings String

Usage

All Entries

On it's own, entering all/ in the search bar will return all reviews that are posted to the database.

Unique List Names

To search for a list of all unique database elements, enter: all/name. This will return all unique elements of that particular field. Example: all/company will return something like: 
[
        {
                "company": "Rhode & Schwarz"
        },
        {
                "company": "Intel Corporation"
        }
]
With this result, we only see the list of unique companies, and if there are many of the same company, only one list item for the set will be displayed. To search for a particular review or set of reviews, we instead turn to the name/ search. Note: for the admin page, /all/name does not work to edited the database. Since /all/name only returns a list, we cannot edit any entries in this manner(refer to filtering entries).

Filtering Entries

To search for specific entries, we can use design the URL to get all of the desired data. The format for this search is: name/value. The name is the field of data such as company or overall_rating. To view all the names of the potential searches, please refer to the table schema. Example: You wanted to find all of the entries for the portland area. You would enter: location/Portland,%20OR (At the moment, all of the searches are case sensitive, so be sure the search will match the result.) This will return: 
[
        {
                "uid": 7,
                "company": "Intel Corporation",
                "location": "Portland, OR",
                "position": "Software Engineering Intern",
                "start": "2020-03-03T08:00:00.000Z",
                "end": "2020-03-27T07:00:00.000Z",
                "intern_type": "other",
                "pay_type": "Annual",
                "pay_amount": "$20,000 - $35,000",
                "work_desc": "",
                "overall_rating": 3,
                "leadership_rating": 3,
                "communication_rating": 3,
                "flexibility_rating": 3,
                "culture_rating": 3,
                "internship_recommend": 3,
                "company_recommend": 3,
                "ratings_desc": ""
        }
]
In this instance, there was currently only 1 entry under the Portland, OR search criteria. There can be many ways to use this feature such as overall_rating/5 to view all of the review entries with an overall rating of 5.
